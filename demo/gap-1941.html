1941 is a half-year wearing a full year's clothes

Temperature, precipitation, and snow records at this station all begin on 16 June 1941 — but the file carries a full 365 rows for that year, and any naive yearly rollup happily reports an annual value computed from 199 days. The result is the hottest year in an 86-year record. It never happened.

How this was checked

Every figure here comes from DuckDB reading data/4354766.csv (32,821 daily rows, 1936-09-01 → 2026-07-11) directly. The two load-bearing queries: 

-- All four variables share one start date
SELECT 'TMAX' v, min(DATE) FROM t WHERE TMAX IS NOT NULL
UNION ALL SELECT 'TMIN', min(DATE) FROM t WHERE TMIN IS NOT NULL
UNION ALL SELECT 'PRCP', min(DATE) FROM t WHERE PRCP IS NOT NULL
UNION ALL SELECT 'SNOW', min(DATE) FROM t WHERE SNOW IS NOT NULL;
--  →  all four: 1941-06-16

-- 365 rows, 199 of them carrying data
SELECT count(*) rows, count(TMAX) days
FROM t WHERE year(DATE) = 1941;
--  →  rows = 365, days = 199

The 166 absent days are present in the file as rows with empty measurement columns, which is why a row count never catches this and a count(TMAX) does. The reconstructed 58.1°F fills 1941's missing window with the 1942–1971 monthly normals and re-averages across a full 365 days; it lands squarely among 1941's neighbours (56.6–59.1°F), whereas the reported 64.5°F does not.

Snow is a partial exception worth stating plainly: 1941 totals 0.0 inches, but 1943 also records 0.0 across the same June–December window, and December 1941's daily rows look ordinary. That zero is a real snowless season, not a second defect — though the annual figure is still wrong, since a normal January–May carries 12.9 inches.
